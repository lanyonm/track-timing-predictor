2. Message States

All messages appear once above the session list. Combined states shown where applicable.

M1 No name active — no message

No messages displayed when no racer name is set.

(no message — schedule renders directly)

M2 Success — matches found with upcoming next race

M3 Success — matches found, next race is single-heat

Name set, matches > 0, next event is single-heat (no "Heat N").

Found 2 event(s) for "Sean Hall" Your next race: Women's Scratch Race at 19:07
M4 Success — matches found, all completed

Name set, matches > 0, but all matched events are completed. No "Your next race" line.

Found 3 event(s) for "Sean Hall"
M5 Success — matches found, next race is currently ACTIVE

Name set, matches > 0, the racer's nearest non-completed event is active (in progress). Uses "Racing now:" label instead of "Your next race:".

Found 2 event(s) for "Sean Hall" Racing now: Women's Individual Pursuit, Heat 3 at 18:43
M6 Success + missing start lists

Matches found, but some events don't have start lists yet. Missing start lists uses amber warning styling.

Found 2 event(s) for "Sean Hall" Your next race: Women's Scratch Race, Heat 2 at 19:07
3 event(s) do not yet have start lists
M7 No matches — start lists exist

Name set, matches == 0, at least one event has a start list.

No matching events found for "Xyz Nobody"
M8 No matches + missing start lists

Name set, matches == 0, some events have start lists + some don't. Both messages use amber warning styling.

No matching events found for "Xyz Nobody"
4 event(s) do not yet have start lists
M9 No data — ALL start lists missing

Name set, ALL events lack start lists. "No matches" suppressed. Amber warning styling.

Start lists are not yet published — check back closer to the event
